## **ğŸ’» Development Workflow**

### **Running in Development Mode**

To start the Vite development server:

pnpm dev

### **Building the Library**

To build the distribution files (ESM and UMD) in the dist/ folder:

pnpm build

### **Running Tests**

We use **Vitest** for testing. Please ensure tests pass before submitting a PR.

pnpm test

### **Linting**

To check for code style issues using ESLint:

pnpm lint

To automatically fix linting issues:

pnpm lint:fix

## **ğŸ› Issues & Feature Requests**

### **Found a Bug?**

* **Ensure the bug was not already reported** by searching on GitHub under [Issues](https://www.google.com/search?q=https://github.com/florianschepp/classy-scroll/issues).  
* If you're unable to find an open issue addressing the problem, open a new one. Be sure to include a **title and clear description**, as well as as much relevant information as possible, and a **code sample** or an executable test case demonstrating the expected behavior that is not occurring.

## **ğŸ“ Pull Requests & Commits**

1. **Create a Branch**: Create a new branch for your feature or fix.  
   * Example: feat/smooth-scroll or fix/typo-in-readme.  
2. **Commit Messages**: We encourage the use of **Conventional Commits**.  
   * feat: ... for new features.  
   * fix: ... for bug fixes.  
   * docs: ... for documentation changes.  
   * chore: ... for build/tooling changes.  
3. **Verify**: Ensure pnpm test and pnpm build pass locally.  
4. **Submit**: Open a Pull Request against the main branch.

## **ğŸ“¦ Release Process (Maintainers Only)**

We use an automated GitHub Actions pipeline to publish to npm.

1. **Update Version**: Bump the version number in package.json (e.g., "version": "1.0.1").  
2. **Commit**: Your commit message **must** contain the word release: to trigger the publication workflow.  
   git commit \-m "release: v1.0.1"

3. **Push**: Push to the main branch.  
   * The CI will run tests and linting.  
   * If successful, it will:  
     * Publish the package to npm.  
     * Create a GitHub Release with the tag (e.g., v1.0.1).
